## 1 引言

<summary><b>点击展开 / 折叠本章内容</b></summary>

### 1.1 标识
- 项目名称：人事管理系统 (HR Management System)
- 项目性质：大学生课程设计项目
- 技术架构：C++11 / Qt 桌面端 (C/S架构)
- 版本号：V1.0

### 1.2 背景
本项目是山东大学计算机科学与技术专业软件工程课程设计要求下提出的。在理论学习阶段结束后，学生需要通过模拟真实企业环境中的业务需求，独立或以团队形式完成一个具有实际应用价值的软件系统。人事管理是任何企业运转的核心，包含人员档案、薪资核算及复杂的统计查询功能，非常适合作为检验学生数据库设计、面向对象编程（C++）及图形界面开发（Qt）能力的综合性课题。

### 1.3 项目概述
本文档适用的项目旨在开发一套结构清晰、功能完备的桌面人事管理系统。系统以关系型数据库为底座，旨在实现员工基础信息、学历、婚姻状况的电子化录入；员工在职/离职状态的动态标记；高度自定义的薪酬计算法则；以及多维度的组合查询与打印输出。本项目由学生团队采用 C++11 标准和 Qt 跨平台框架进行开发，最终成果将作为期末考核及项目实践的重要依据。

### 1.4 文档概述
本文档作为课程设计初期的规划核心，主要用途是深度解析人事管理系统的业务规则和功能要求，并重点从开发成本、实践效益、技术实现路径等方面进行可行性论证。

## 2 可行性分析的前提
<details>
<summary><b>点击展开 / 折叠本章内容</b></summary>

### 2.1 项目的要求
基于课程设计的题目要求，系统必须满足以下核心功能指标：
- 全维度信息录入与修改：系统需支持员工各种信息的结构化输入与修改，字段至少涵盖：基本信息（姓名、年龄等）、学历信息、婚姻状况信息、职称与工作岗位等。
- 生命周期与状态标记：系统需具备严密的状态管理机制，对于转出、辞职、辞退、退休的员工信息需做相应的独立标记。在默认的数据视图中，仅显示“在职”员工信息，离职人员进入历史档案库供追溯。
- 高灵活度薪酬自定义：打破固定公式，工资、奖金、扣款等细节必须可以由 HR 角色进行自定义配置。系统需生成电子工资条，且支持员工通过客户端登录查看个人的薪资明细。
- 多维查询、统计与打印：
  - 查询功能：支持按条件精准或模糊查询，至少包括：查询员工详细信息、按婚姻状况查询、按学历查询、按工作岗位查询等。
  - 统计功能：支持数据的宏观统计归类，至少包括：按学历、婚姻状况、岗位、参加工作时间、工资等维度统计对应的人数或金额占比。
  - 输出功能：对所有的查询和统计结果，系统必须支持直接打印输出或导出报表。

### 2.2 项目的目标
- 学术与技能目标：将课堂所学的 C++ 面向对象思想、Qt 信号与槽机制、数据库范式设计融会贯通，产出一段高质量、规范化的软件工程代码。
- 业务模拟目标：构建一个逻辑闭环的人事管理原型系统，界面友好，数据不冲突，完美演示真实企业中 HR 部门核心业务流转的全过程。

### 2.3 项目的环境、条件、假定和限制
- 开发与运行环境：利用学生自有的个人电脑（Windows 或 macOS）进行开发。假定使用 Qt Creator 配合 MinGW/MSVC 编译器，数据库使用轻量级的 SQLite 以方便系统演示。
- 时间与人力限制：项目属于课程设计，开发周期根据课程需求而定，开发人员为学生小组。要求在有限时间内聚焦核心功能（档案增删改查、薪资计算、统计打印），不要求实现企业级的高并发集群部署。

### 2.4 进行可行性分析的方法
鉴于本项目为课程设计，采用以下论证方法：
1. 时间-任务分解法：评估在课设规定周期内能否完成指定的代码量和系统模块。
2. 能力边界评估法：将需求拆解为具体的技术点（如 SQL多条件拼接查询、Qt打印机模块调用），评估小组成员的技术储备是否匹配。
3. 综合成本效益分析：从资源消耗与教育实践价值的角度衡量项目的投入产出合理性。
</details>

## 3 可选的方案
<details>
<summary><b>点击展开 / 折叠本章内容</b></summary>

### 3.1 原有方案（纯手工/传统Excel管理）的局限性
在论证新系统的必要性之前，首先需要剖析模拟业务场景中的“原有方案”。假设企业目前采用传统的纸质档案结合手工 Excel 表格进行人事与薪资管理，这种原有方案存在以下显著的痛点与局限性：
- 数据孤岛与一致性极差：员工的基础档案、每月的考勤记录、以及薪资发放明细通常散落在不同 HR 人员的本地电脑中。数据更新不同步，极易导致“数据打架”、版本混乱和信息冗余。
- 薪资核算效率低下且易出错：高度依赖人工核对公式来计算薪资和扣款，工作量巨大且枯燥。一旦 Excel 公式被误改或数据错位，容易引发连锁计算错误；同时，人工操作缺乏审计日志，无法追溯数据的修改历史。
- 安全与隐私风险高：传统的 Excel 文件极易被通过 U盘或网络工具随意复制、外传。员工的敏感隐私（如具体薪水数字、身份证号、家庭情况）暴露在文件系统中，缺乏基于用户角色的细粒度权限隔离保护。
- 缺乏自动化统计与报表能力：面对“统计某学历区间的人数占比”或“查询某时间段的新员工入职率”等宏观决策需求时，需耗费大量人工时间进行数据筛选和透视表制作，无法做到一键生成直观图表或直接连接打印机规范输出。

### 3.2 课设常见的可选方案及优缺点对比
针对上述原有方案的局限性，在确定课程设计的技术升级栈时，通常有以下几种方案：
- 方案A：基于控制台 (Console) 的 C/C++ 系统
  - 优点：极简，零 UI 负担，纯粹考核底层逻辑代码。
  - 缺点：过于简陋，交互极其繁琐，无法直观展示复杂的数据报表（尤其是题目要求的表格统计和打印输出），功能呈现效果存在较大局限。
- 方案B：基于 Java Web (Spring Boot + Vue) 架构系统
  - 优点：符合当前互联网主流开发模式，界面美观。
  - 缺点：对于主修 C++ 的课程而言属于偏题；前后端分离架构搭建繁琐，对于短短几周的课设而言，环境配置和学习成本偏高；且 Web 页面直接调用本地打印机实现精美排版相对困难。
- 方案C：基于 C++11 和 Qt 的桌面 GUI 系统
  - 优点：完美契合 C++ 课程要求。Qt 提供了极其强大的界面拖拽设计器（Qt Designer）、开箱即用的数据库模块（Qt SQL）以及原生的高级打印支持（Qt Print Support）。使用 QTableView 可直接与数据库表模型绑定，非常适合展示“员工表”、“工资条”等多维数据。
  - 缺点：需要额外学习 Qt 的信号与槽机制及框架 API。

### 3.3 选择最终方案的准则
综合考虑“解决原有方案数据混乱痛点、课程知识巩固、功能贴合度（特别是报表统计与打印）、时间紧迫性及呈现效果”，选择 方案C（C++11/Qt 桌面架构） 作为最终方案。它不仅能在系统演示时提供直观、原生的桌面窗口体验，还能最优雅地实现复杂的本地数据处理与外设打印交互，彻底克服了人工 Excel 方案的各项局限。
</details>

## 4 所建议的系统
<details>
<summary><b>点击展开 / 折叠本章内容</b></summary>

### 4.1 对所建议的系统的说明
本系统是一个基于 C/S（Client/Server）模式的轻量级桌面应用程序。系统采用 MVC/Model-View 架构，将数据存储（SQLite数据库）、业务逻辑处理（C++11类对象）与图形界面展示（Qt Widgets）进行有效分离，确保代码的高内聚与低耦合。

### 4.2 系统核心模块设计
根据原始需求，系统将被划分为四大核心子系统模块：
- 核心模块一：员工档案动态管理子系统
  负责员工全量信息的增删改查。系统界面将提供分类表单（基础区、学历区、岗职区）供 HR 录入信息。系统底层自动关联员工编号，实现信息的结构化存储。
- 核心模块二：人员状态流转子系统
  此模块独立于基础信息，专门维护员工的生命周期标记。在执行“转出/辞退/退休”操作时，系统不会物理删除数据库记录，而是更改其状态标识位（如 status 字段）。并在全局数据视图中设置默认过滤器，屏蔽非在职人员，确保 HR 日常操作面板的整洁，同时保留数据追溯的可能。
- 核心模块三：自定义薪酬核算引擎
  提供一个“薪资规则配置”界面。HR 可以动态添加或修改薪资计算项。在月末结转时，C++ 后台通过读取这些自定义规则，自动遍历在职员工生成当月实发工资，并生成对应的电子工资条记录供员工端查询。
- 核心模块四：多维统计与报表输出模块
  通过拼接动态 SQL 语句，实现复杂的组合查询（例如 WHERE education='本科' AND marital_status='已婚'）。系统利用 Qt 的绘图与表格组件，将统计结果直观呈现，并直接调用 QPrinter 和 QPrintDialog 类，支持将结果导出为 PDF 或直接发送至物理打印机。

### 4.3 系统的业务处理流程
- 数据初始化流：管理员配置系统基础字典 -> HR 进行员工档案建档并设定默认状态为“在职”。
- 日常管理流：HR 根据业务变动，进入系统修改员工信息（如学历晋升、婚姻状态变更）或变更员工状态（办理离职）。
- 薪资与自助查询流：HR 自定义本月奖惩明细 -> 系统一键生成全员本月薪水账单 -> 普通员工在自己的电脑上运行客户端，输入工号密码登录 -> 界面仅展示该员工的个人信息页和其专属的电子工资条视图。
</details>

## 5 经济可行性(成本----效益分析)
<details open>
<summary><b>点击展开 / 折叠本章内容</b></summary>

鉴于本项目为计算机专业课程设计，其经济可行性评估有别于企业级商业项目。本部分主要从项目的开发资源消耗、学习资源转化以及综合教育效益的角度进行论证：

### 5.1 开发成本分析
本系统的开发主要依赖于现有的教学环境与开源技术生态，直接的资金成本极低：
- 硬件资源成本：系统的开发与测试运行完全可依托于学生现有的个人电脑。由于系统定位为轻量级桌面端应用，对 CPU、内存及存储的算力要求较低，无需购置专用的服务器环境。
- 软件授权成本：项目全程采用免费且成熟的开源技术栈。集成开发环境使用 Qt Creator 开源版，底层数据库采用轻便的单机文件型关系数据库 SQLite。因此，软件正版授权及部署的财务支出为零。
- 时间与人力投入：团队成员需在规定的课设周期内投入时间进行需求分析、编码、调试与文档撰写。这部分隐性的时间成本是课程实践的必要组成部分，其本质是将时间转化为专业知识的累积过程。

### 5.2 预期综合效益
项目的核心产出不仅是一个可运行的程序，更体现在技术验证、知识实践与综合能力的提升上：
- 理论与实践的深度融合：通过完整的软件生命周期开发，学生能够将抽象的 C++ 面向对象编程理论、关系型数据库设计范式（如主外键关联、SQL查询）以及复杂图形界面（GUI）的事件驱动模型，切实转化为直观可用的实体软件。
- 架构选型的技术验证：本项目将验证“C++11 + Qt + SQLite”这一经典技术组合在处理人事业务管理、多维数据统计与本地报表生成时的轻量化与高效性，为后续设计更复杂的管理系统积累了坚实的架构经验。
- 系统原型的应用价值：系统产出是一个逻辑闭环、功能清晰的通用人事业务原型。它能够有效地模拟企业员工从入职录入、动态状态追踪到薪资核算的全过程，具备一定的业务示范和教学研讨意义。

### 5.3 投入产出评估与风险控制
- 综合投入产出分析：本项目以极低的软硬件资源消耗，换取了对软件工程完整流程的实战经验和编程技能的实质性强化，项目的综合教育投入产出比非常合理。

### 5.4 后续衍生价值
系统采用了清晰的模块化与 MVC 架构设计，代码结构具有良好的可读性与可维护性。在课程结束后，该项目具备持续扩展的潜力。

## 6 技术可行性(技术风险评价)
<details>
<summary><b>点击展开 / 折叠本章内容</b></summary>

### 6.1 技术选型的成熟度与匹配度
- C++11 核心特性的应用：本项目抛弃老旧的 C 语言风格，全面拥抱现代 C++。利用 STL 容器（如 std::vector, std::map）处理在内存中的统计数据分组；利用智能指针（std::shared_ptr）管理复杂的界面对象，从根本上降低内存泄漏风险，提升开发效率。
- Qt 框架的完美赋能：
  - 数据展示：Qt 的 QSqlTableModel 可以直接将 SQLite 数据表映射到界面的 QTableView 控件中，只需几行代码即可实现具备排序、分页功能的漂亮数据表格。
  - 打印与导出：这是本项目区别于普通课设的一大亮点。利用 Qt 的 Qt Print Support 模块，系统可以将分析统计好的图表（基于 Qt Charts）或人员名单文本，一键渲染输出为 PDF 报告或直接调取 Windows/Mac 本地的物理打印机，完美解决需求中对“输出功能”的硬性规定。

### 6.2 难点与技术风险应对
- 难点1：自定义薪资计算公式的实现。
  - 应对策略：传统的课设通常把工资计算写死在代码里。为满足“细节自定义”要求，可以在数据库中建立一张“薪资模板表”存储各项金额的基数，C++ 后端在运算时动态读取这些配置进行求和与抵扣，从而实现薪酬结构的解耦与高灵活性。
- 难点2：演示环境跨机运行奔溃风险。
  - 应对策略：使用本地 MySQL 服务器容易在换电脑演示时因为环境不一致而跑不起来。项目将强制采用 SQLite 文件型数据库，将后缀为 .db 的数据库文件连同程序一并打包。并且利用 Qt 的 windeployqt 工具提取所有必须的 DLL 动态链接库，实现程序的“绿色免安装”双击运行，彻底规避跨机运行的环境兼容性风险。
- 结论：所选技术栈均为成熟的教学经典方案，且针对常见风险均有完备的避险策略，技术难度完全在大学生的合理能力边界内，技术可行性充分通过。
</details>

## 8 用户使用可行性
<details>
<summary><b>点击展开 / 折叠本章内容</b></summary>

作为演示评估项目，系统的主要“终端用户”是指导老师和进行模拟操作的同学。
- Qt 能够生成极其原生的桌面窗口界面，系统将采用大家熟悉的顶部菜单栏、左侧导航树和中央数据表格的经典布局，完全符合日常 Windows/Mac 办公软件的操作直觉。
