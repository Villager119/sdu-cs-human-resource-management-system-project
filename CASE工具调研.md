# 一、可视化 & UML 建模工具
## 1. Microsoft Visio
### 用途
- 绘制流程图、组织结构图、网络拓扑图、ER 图、UML 简图、BPMN 图等
- 用于需求文档、系统设计草图、汇报演示、流程梳理
- 快速可视化表达系统结构与业务流程

### 技术特点
- 微软 Office 生态深度集成，界面简单易用
- 海量内置模板与形状库，拖拽式操作
- 支持矢量图、自动对齐、智能连接线
- 可导出 PDF、PNG、SVG 等多种格式
- 支持简单数据绑定与实时协作

### 缺点
- 不是专业 UML / 建模工具，UML 语法校验弱
- 不支持代码生成、逆向工程、模型驱动开发
- 复杂大型建模能力差，协作与版本管理较弱
- 商业软件，付费订阅，价格较高

### 应用场景
- 业务人员梳理流程
- 架构师画系统草图、网络拓扑
- 项目文档配图、PPT 汇报图表
- 非专业建模人员快速出图

---

## 2. Rational Rose
### 用途
- 完整支持 UML 1.x 标准建模（用例图、类图、时序图、状态图等）
- 支持 C++、Java、Smalltalk 等语言的代码正向 / 逆向工程
- 用于传统软件工程、面向对象设计、详细设计阶段建模

### 技术特点
- 早期最经典、最主流的 UML CASE 工具
- 模型与代码双向同步，支持模型驱动开发
- 支持团队协作、模型版本管理、Rose 模型文件存储
- 与 IBM Rational 工具链（ClearCase、ClearQuest）深度集成

### 缺点
- 界面老旧，操作繁琐，学习成本高
- 仅支持 UML 1.x，不支持 UML 2.0+ 新标准
- 性能差，大型模型卡顿严重
- 已基本停止更新，被 IBM Rational Software Architect 替代
- 商业授权昂贵

### 应用场景
- 传统大型企业遗留项目维护
- 高校 UML 教学、老式软件工程课程
- 依赖旧版 Rose 模型的 legacy 系统设计

# 二、版本控制与配置管理工具
## 3. Git
### 用途
- 管理源代码版本、追踪修改历史
- 支持多人并行开发、分支管理、代码合并
- 实现代码回滚、发布版本管理、协作开发

### 技术特点
- 分布式：每个开发者拥有完整仓库，不依赖中央服务器
- 强大分支模型：轻量创建、快速切换、高效合并
- 哈希校验保证完整性，本地操作速度极快
- 支持 GitHub/GitLab 等远程仓库，跨地域协作
- 开源免费，社区生态极其丰富

### 缺点
- 命令行概念较多，新手学习曲线较陡
- 二进制大文件管理效率一般（需 LFS）
- 复杂合并冲突时解决难度较高
- 权限控制较简单，不适合超严格企业管控

### 应用场景
- 互联网公司、开源项目、中小团队开发
- 敏捷开发、持续集成 CI/CD
- 跨地域团队协作
- 几乎所有现代软件开发场景

---

## 4. ClearCase
### 用途
- 大型团队、复杂项目的代码与文档版本控制
- 严格的权限管理、配置项管理、基线管理
- 支持并行开发、版本追溯、变更控制
- 常用于航空、军工、金融等强合规行业

### 技术特点
- 集中式架构，强大的权限与访问控制
- 支持精细版本管理、组件化配置管理
- 与 Rational Rose、ClearQuest 缺陷管理工具联动
- 支持复杂基线、多分支、多平台环境管理
- 极高稳定性，适合超大型长期项目

### 缺点
- 极其笨重，操作复杂，学习成本极高
- 速度慢，服务器压力大，本地操作依赖网络
- 价格极其昂贵，部署维护成本高
- 生态封闭，与现代工具链集成差
- 社区萎缩，逐渐被 Git 替代

### 应用场景
- 军工、航天、金融、大型传统企业
- 对合规、审计、权限有极高要求的项目
- 历史遗留的巨型软件工程

---

# 三、代码分析与逆向工程工具
## 5. CodeLogic
### 用途
- 自动分析源代码之间的调用关系、依赖关系、数据流
- 从复杂系统中自动生成类图、时序图、架构图
- 帮助理解遗留系统、做技术债务梳理、变更影响分析
- 支持数据库与代码之间的关联追踪

### 技术特点
- 无需手动建模，自动扫描解析代码
- 支持 Java、C#、Python、JavaScript 等多语言
- 可视化展示调用链、依赖图、数据库关联
- 提供变更影响分析：修改一处代码会影响哪些模块
- 可与 IDE、CI/CD 集成，辅助架构治理

### 缺点
- 商业软件，授权费用高
- 超大规模代码库分析可能较慢
- 生成图表偏自动化，不够美观，需手动调整
- 主要偏向分析，不支持正向设计与建模

### 应用场景
- 遗留系统重构、接手陌生大型项目
- 架构审计、技术债务治理
- 变更风险评估、影响范围分析
- 复杂微服务 / 单体系统架构梳理
